How We're Grading

Three states, not two.

Most self-assessments binary-grade capability as "built vs. not built." That hides the real question: an idea can be codified without being wired, and wired without being activated. JIRAH's gap is not knowledge. It is the distance between pages 2 and 3.

Every capability on the following slides is scored against these three states, which matters because 38 skills on disk is a very different sentence from 38 skills with production mileage.

01
Codified

The IP exists on disk — skill file, context doc, subagent. A competent operator could pick it up and run it cold.

02
Wired

Integrated into the stack — dashboard reads it, skill composes it, subagent fans out on it. Present at the right moment.

03
Activated

Has battle mileage. Ran in a real engagement or on a real cadence. The feedback loop has closed at least once.

What's Not Built Yet

Six gaps, one of them load-bearing.

No euphemisms. Each of these is a real absence that shows up when we ask "could we onboard a third partner tomorrow" or "could we run at 40 prospects instead of 4."

Load-bearing
Zero scheduled triggers are wired.

The daily briefing, Monday weekly review, Friday harvest scan, monthly KPI — the cadence skills exist, but no cron runs them. Right now both partners run skills manually. This is the gap between a machine and a playbook-in-a-drawer.

Integration
MCPs not wired: Outlook Mail, Calendar, Fireflies API, Canva.

Skills draft emails, scan transcripts, and describe Canva work — but they don't send, fetch, or render. Human sits in the middle of every handoff. High-volume scenarios will overload the partner before the machine.

Deployment
Dashboard is localhost-only.

Each partner runs npm run dev on their own machine. State is synced via OneDrive + Git. No browser-accessible URL, no mobile access, no Vercel deployment. Works for two partners; breaks with any third viewer (advisor, client-side stakeholder, auditor).

Mileage
~34 of 38 skills have never run in anger.

They're codified. They're wired. They have not met a real engagement. The next real Discovery call will be the first time most of them hit friction with a live client and a real transcript. Expect breakage; build feedback loops for rapid iteration.

Observability
No telemetry · no invocation history · no logging.

We cannot answer "which skills have been called, when, with what inputs, producing what outputs?" That means no learning loop yet. Every improvement relies on partner memory instead of recorded signal.

Scale
No client-facing AI employee pattern to re-sell.

Shop-Drawing is the only exemplar. We can cite it, not clone it. A second and third named AI-employee pattern (e.g., Partnership Proposal Generator, Market Expansion Evaluator) would turn the AI service line from bespoke into productized. Directly echoes the bootcamp memo's P3 recommendations.

The Growth Pathway

Three horizons. Each unlocks the next.

The order matters. Activating the machine (H1) is the prerequisite for scaling the service line (H2), which is the prerequisite for broadcasting the platform (H3). Skip H1 and the rest are forecasts, not plans.

Horizon 1 · Next 30 Days

Activate the Machine.

Move skills from codified to activated by wiring cadence and closing one engagement end-to-end.

  • Wire the daily briefing trigger — single cron on Jason's dev machine, proves the pattern, unlocks the rest of the 8-trigger stack
  • Run one prospect end-to-end through Discovery → Discussion Doc → Kickoff. Break things on purpose; update skills from feedback
  • Add self-scoring gate to /jirah-report-review (bootcamp memo P1)
  • Activate /jirah-industry-pulse on a weekly trigger — appends straight to prospects/
  • Record one invocation log file — simple append-only trace of which skill ran with which inputs. Seed of telemetry
Horizon 2 · 30–90 Days

Ship the Client Business Brain.

Upgrade every AI pilot from one-off build to compounding per-client asset. Productize a second exemplar.

  • Wire Outlook Mail + Calendar MCP — skills send, book, and follow up. Eliminates the human-in-the-loop tax
  • Ship the Client Business Brain as a standard pilot-plan line item (bootcamp P1 #1) — retrofit S+A as Test 1
  • Decide + adopt "AI Employee" language (bootcamp P1 #3) — update jirah-voice.md once
  • Test the per-employee $6–10K/mo tier on S+A before external pitch
  • Productize exemplar #2 — Partnership Proposal Generator or Market Expansion Evaluator, only when a real ICP prospect pulls it forward
  • Compliance-context file per vertical (engineering, accounting, pro services) — unlocks vertical pilot scaling
Horizon 3 · 6+ Months

Broadcast the Platform.

Move from internal tool to partner-visible platform. Open the machine to a third party without inviting chaos.

  • Deploy dashboard to Vercel — browser-accessible, auth-gated, shared-state via MS Graph or Supabase bridge
  • Real telemetry layer — invocation history, error tracking, learning-loop reports fed back to skill refinement
  • Cloud-scheduler infrastructure — Azure Functions or AWS EventBridge for cadence, ready to drop into client pilots too
  • First public case study — S+A or Genesis, editorial long-form, published via /jirah-case-study
  • AI Architect retainer lane as a named Step 6 tier, distinct from ops advisory
  • Pattern library with ≥ 8 engagements — quantifies the wedge; feeds /jirah-win-loss
