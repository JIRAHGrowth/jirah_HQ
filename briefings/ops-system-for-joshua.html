01   What It Is

A Claude-managed workspace that acts as Jirah's command center.

JIRAH Ops lives on my desktop in a single folder. When I open Claude Code inside that folder, Claude loads all of Jirah's IP, process, and voice rules automatically. It is not a generic AI assistant. It is a Jirah-aware one.

The system drives the full growth loop end to end: cold lead, Friction Audit, closed client, delivered engagement, case study, referral. Everything we do as a firm has a place in it.

Claude Code Chat

For drafting, diagnosing, generating. You type slash commands and Claude produces deliverables in Jirah's voice.

Surface 2

05   How To Use It

Two surfaces. Use them together.

Surface 1

Claude Code Chat · For drafting, diagnosing, generating

Open Claude Code in the workspace folder. Claude auto-loads the full Jirah context stack: firm IP, voice rules, the 10 Friction Points, the 6-step process, visual modes, pattern library.

You invoke work with slash commands. Type /jirah-daily-briefing and Claude scans pipeline, engagements, and the audit queue, then writes today's briefing.

Surface 2

Local Dashboard · For seeing state at a glance

Run cd dashboard && npm run dev and open localhost:3000. Five views:

  • Today's Focus Bar. Overdue follow-ups, audit queue count, engagements at risk, content-this-week status.
  • Pipeline Kanban. Cold / Warm / Meeting Booked / Proposal Sent / Closed Won / Closed Lost.
  • Audit Applications Queue. Sortable table with ICP score, status, triage action.
  • Active Engagements. Card grid with a 6-step stepper per client.
  • Today's Briefing Panel. Renders the day's briefing markdown inline.

09   For Joshua

What you do first.

Six steps to get your mirror running. Should take under an hour.

  • Clone the workspace. Copy c:\Users\jason\Desktop\Jirah Ops\ to the same path on your machine. Same path matters for the permissions config.
  • Install Claude Code. If you do not have it yet. Sign in with your Anthropic account.
  • Verify the context loaded. Open Claude Code in the workspace. Ask: "what does Jirah do?" The answer should be Jirah-specific, not generic. If it is generic, the context did not load.
  • Spin up the dashboard. Run cd dashboard && npm install, then configure .env.local with your OneDrive path, then npm run dev. Open localhost:3000.
  • Start your mornings with /jirah-daily-briefing. It writes briefings/YYYY-MM-DD.md. Read that before anything else.
  • Treat ghost skills as specs, not errors. Invoke one and you get a build brief back. That brief is what we fill in when the skill is first needed in live work. Ship when needed, not before.
